# Show Types

This catalog is the public index of show types Show Sidekick can create. It is also the source of truth for show-type validation: every row has a stable `lane_id` that validation tools can resolve against bundled pipeline and starter inventory.

Pipeline rows are reusable workflow types from `bundled/pipelines/*.yaml`. Starter rows are complete starter shows from `bundled/starters/*`. A branded concept such as The Chaos FM, WW2 Diary, Product Demo, Last Rev, Rave Queen, or AI Workflow Demo is a starter lane, not a pipeline slug.

## Local Validation

Run the non-rendering catalog check before promising a show type publicly:

```bash
pnpm show-types:check
```

Run the catalog-driven matrix from the harness checkout before publishing. It creates fresh temp user projects outside the repo, runs one representative sample for each runnable lane in the selected mode, and writes `show-types-matrix-report.json` plus `show-types-matrix-report.md` in the matrix work directory.

```bash
pnpm show-types:matrix --zero-key
pnpm show-types:matrix --paid-demo
```

The per-lane status values are `not-run`, `unsupported`, `setup-missing`, `build-failed`, `export-failed`, and `verified`. Unsupported lanes remain documented so operators can see which public show types still need a runnable sample path.
